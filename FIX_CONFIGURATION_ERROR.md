# Исправление ошибки "Configuration"

## Проблема:
Ошибка "Configuration" в NextAuth, связанная с несоответствием портов.

## Решение:

### Шаг 1: Добавьте NEXTAUTH_URL в .env.local

Откройте файл `.env.local` и добавьте (или обновите) строку:

```env
NEXTAUTH_URL=http://localhost:3001
```

**ВАЖНО:** Используйте тот порт, на котором работает ваш сервер (3001, если сервер запущен на 3001).

### Шаг 2: Перезапустите dev-сервер

1. Остановите текущий сервер (`Ctrl+C`)
2. Запустите снова:
   ```powershell
   npm run dev
   ```

### Шаг 3: Используйте правильный URL

Убедитесь, что вы открываете правильный порт в браузере:
- Если сервер на порту 3001: `http://localhost:3001/login`
- Если сервер на порту 3000: `http://localhost:3000/login`

## Проверка:

После добавления `NEXTAUTH_URL` и перезапуска сервера:
1. Откройте правильный URL в браузере
2. Нажмите "Войти через Google"
3. Ошибка "Configuration" должна исчезнуть

## Если проблема сохраняется:

1. Проверьте, что `NEXTAUTH_URL` соответствует порту сервера
2. Проверьте, что все переменные окружения заполнены в `.env.local`
3. Проверьте логи сервера на наличие ошибок
