# Как пересобрать проект на Vercel

## Способ 1: Через Vercel Dashboard (Рекомендуется)

1. **Откройте Vercel Dashboard:**
   - Перейдите на [https://vercel.com/dashboard](https://vercel.com/dashboard)
   - Войдите в свой аккаунт

2. **Выберите ваш проект:**
   - Найдите проект "LifeScript" (или ваше название проекта)
   - Нажмите на него

3. **Перейдите в Deployments:**
   - В верхнем меню выберите вкладку **"Deployments"**
   - Вы увидите список всех деплоев

4. **Найдите последний деплой:**
   - Обычно это самый верхний в списке
   - Наведите курсор на деплой

5. **Запустите Redeploy:**
   - Нажмите на **"..."** (три точки) справа от деплоя
   - Выберите **"Redeploy"**

6. **Настройки Redeploy:**
   - Появится диалоговое окно
   - **ВАЖНО:** Снимите галочку **"Use existing Build Cache"** (или установите в "No")
   - Это заставит Vercel пересобрать проект с нуля, используя новые переменные окружения
   - Нажмите **"Redeploy"**

7. **Дождитесь завершения:**
   - Процесс займет 1-3 минуты
   - Вы увидите прогресс сборки в реальном времени
   - После завершения проект будет доступен по вашему URL

## Способ 2: Через Git (Автоматический)

Если ваш проект подключен к Git (GitHub, GitLab, Bitbucket):

1. **Сделайте пустой коммит:**
   ```powershell
   git commit --allow-empty -m "Пересборка для применения переменных окружения"
   ```

2. **Отправьте изменения:**
   ```powershell
   git push
   ```

3. **Vercel автоматически:**
   - Обнаружит новый коммит
   - Запустит сборку
   - Задеплоит новую версию

## Способ 3: Через Vercel CLI

Если у вас установлен Vercel CLI:

1. **Установите Vercel CLI (если еще не установлен):**
   ```powershell
   npm i -g vercel
   ```

2. **Войдите в Vercel:**
   ```powershell
   vercel login
   ```

3. **Задеплойте проект:**
   ```powershell
   vercel --prod
   ```

## Когда нужно пересобирать проект:

✅ **Обязательно пересобирайте после:**
- Добавления новых переменных окружения
- Изменения существующих переменных окружения
- Изменения `vercel.json`
- Изменения `package.json` (новые зависимости)
- Изменения конфигурации сборки

⚠️ **Не обязательно пересобирать после:**
- Изменения только кода (если проект подключен к Git - деплой произойдет автоматически)
- Изменения только в `.env.local` (это локальный файл, не влияет на Vercel)

## Проверка после пересборки:

1. **Проверьте статус деплоя:**
   - В Vercel Dashboard → Deployments
   - Убедитесь, что последний деплой успешен (зеленая галочка)

2. **Проверьте работу приложения:**
   - Откройте ваш Vercel URL
   - Проверьте, что приложение работает корректно

3. **Проверьте переменные окружения:**
   - Vercel Dashboard → Settings → Environment Variables
   - Убедитесь, что все переменные на месте

## Troubleshooting:

### Если деплой не запускается:
- Проверьте, что проект подключен к Git репозиторию
- Проверьте, что у вас есть права на проект

### Если деплой падает с ошибкой:
- Проверьте логи деплоя (Vercel Dashboard → Deployments → последний деплой → Logs)
- Ищите ошибки, связанные с:
  - Переменными окружения
  - Зависимостями
  - Сборкой проекта

### Если изменения не применяются:
- Убедитесь, что вы сняли галочку "Use existing Build Cache"
- Попробуйте очистить кэш: Vercel Dashboard → Settings → General → Clear Build Cache
