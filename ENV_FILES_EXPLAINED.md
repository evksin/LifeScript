# Разница между .env и .env.local в Next.js

## Приоритет загрузки переменных окружения:

Next.js загружает переменные окружения в следующем порядке (последний имеет приоритет):

1. `.env` - общие переменные для всех окружений
2. `.env.local` - локальные переменные (переопределяют `.env`)
3. `.env.development` / `.env.production` - для конкретного окружения
4. `.env.development.local` / `.env.production.local` - локальные для конкретного окружения

## Рекомендации:

### ✅ Правильный подход:

**`.env.local`** - используйте для:
- Реальных секретных данных (пароли, ключи API)
- Локальных настроек разработки
- Данных, которые не должны попасть в Git

**`.env`** - используйте для:
- Примеров значений (с пустыми или тестовыми данными)
- Общих настроек, которые можно закоммитить
- Документации переменных

### ❌ Не рекомендуется:

- Хранить реальные секреты в `.env` (если он может попасть в Git)
- Дублировать одинаковые данные в оба файла

## В вашем случае:

Если у вас одинаковая информация в обоих файлах:

1. **Это не критично** - оба файла в `.gitignore`, так что секреты не попадут в Git
2. **Но лучше оставить только `.env.local`** - это стандартная практика для локальной разработки
3. **Можно удалить `.env`** или оставить его с примерами (без реальных секретов)

## Что делать:

### Вариант 1: Оставить только `.env.local` (Рекомендуется)

1. Удалите файл `.env`
2. Используйте только `.env.local` для всех переменных

### Вариант 2: Использовать оба файла правильно

1. **`.env`** - оставьте с примерами:
   ```env
   DATABASE_URL="postgresql://user:password@host:5432/database?sslmode=require"
   GOOGLE_CLIENT_ID="your-client-id-here"
   GOOGLE_CLIENT_SECRET="your-client-secret-here"
   AUTH_SECRET="your-auth-secret-here"
   ```

2. **`.env.local`** - реальные значения:
   ```env
   DATABASE_URL="postgresql://реальный-connection-string"
   GOOGLE_CLIENT_ID="реальный-client-id"
   GOOGLE_CLIENT_SECRET="реальный-client-secret"
   AUTH_SECRET="реальный-auth-secret"
   ```

## Важно:

- ✅ Оба файла в `.gitignore` - секреты не попадут в Git
- ✅ `.env.local` имеет приоритет над `.env`
- ✅ Для Vercel используйте Environment Variables в Dashboard (не файлы `.env`)

## Проверка:

Убедитесь, что оба файла в `.gitignore`:

```gitignore
.env*.local
.env
```

Это уже настроено в вашем проекте! ✅
