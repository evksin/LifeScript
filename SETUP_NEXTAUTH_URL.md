# Настройка NEXTAUTH_URL для порта 3000

## Проблема:
Сервер работает на порту 3000, но NextAuth может неправильно определять baseUrl, что вызывает ошибки при авторизации.

## Решение:

### Шаг 1: Откройте файл .env.local

### Шаг 2: Добавьте или обновите строку:

```env
NEXTAUTH_URL=http://localhost:3000
```

**ВАЖНО:** 
- Используйте `http://` (не `https://`)
- Используйте `localhost` (не `127.0.0.1`)
- Используйте порт 3000 (если сервер на 3000)

### Шаг 3: Перезапустите dev-сервер

1. Остановите сервер: `Ctrl+C` в терминале
2. Запустите снова:
   ```powershell
   npm run dev
   ```

### Шаг 4: Проверьте работу

1. Откройте `http://localhost:3000/login`
2. Нажмите "Войти через Google"
3. Должен произойти редирект на Google для авторизации

## Если проблема сохраняется:

1. Проверьте, что в `.env.local` нет лишних пробелов вокруг `=`
2. Проверьте, что файл `.env.local` сохранен
3. Убедитесь, что сервер перезапущен после изменения `.env.local`
